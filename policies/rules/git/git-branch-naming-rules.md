# Git Branch Naming Rules
1. Use lowercase only
2. Use hyphens (-) to separate words (not spaces or underscores)
3. Prefix with type:
- feature/ → new feature
- bugfix/ or fix/ → bug fix
- hotfix/ → urgent production fix
- refactor/ → code improvement
- chore/ → maintenance
- docs/ → documentation
4. Include short, clear description (what the branch does)
5. Optionally include issue/task ID (for tracking)
6. Keep it concise but meaningful
7. Avoid generic names (test, new, temp)
8. One branch = one purpose
